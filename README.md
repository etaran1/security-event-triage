# Security Event Triage & Incident Reporting (SOC Analyst I–II)

## Goal
Demonstrate SOC-level triage: detect → assess → document → escalate/close.

## What this repo contains
- Sample log events (sanitized) for triage practice
- A triage decision flow (severity + actions)
- An incident report template
- Example completed incident reports

## How to use
1. Review logs in `data/sample_events.jsonl`
2. Use `docs/triage_flow.md` to classify and decide actions
3. Write your findings using `templates/incident_report.md`
4. Compare to `examples/` for reference

## Skills demonstrated
- Alert triage logic
- Incident documentation quality
- Escalation decision making
- Clear communication for handoff
